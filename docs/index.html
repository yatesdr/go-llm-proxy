Web search works by intercepting server-side search tool calls, executing them via Tavily, and injecting results back into the conversation. Vision pipeline routes images to a vision-capable model for description when the target model is text-only.

Quick start

Create a config.yaml with your models and keys:

listen: ":8080" models: - name: my-model backend: http://192.168.1.10:8000/v1 # vLLM, llama-server, Ollama, etc. keys: - key: sk-your-secret-key name: admin

Run it:

# Docker docker compose up -d # Or binary ./go-llm-proxy -config config.yaml

Point any OpenAI or Anthropic-compatible client at http://localhost:8080/v1. That's it.

The built-in config generator at GET / creates ready-to-use configs for each coding agent. Enable it with -serve-config-generator or serve_config_generator: true in config.

Features

Protocol translationAnthropic Messages API ↔ Chat Completions. OpenAI Responses API ↔ Chat Completions. Auto-detected per backend.
Model multiplexingMultiple models across multiple backends. Name rewriting, per-model timeouts, per-key model restrictions.
Vision pipelineImages sent to text-only models are described by a vision-capable model and replaced with text. Transparent to the client.
Web searchServer-side search tools (Claude Code, Codex) executed at the proxy via Tavily. Works in both streaming and non-streaming modes.
API key managementIssue proxy keys with per-key model restrictions. Backend credentials stay on the server.
Usage monitoringPer-request logging to SQLite. Token counts, latency, per-user breakdowns. Web dashboard and CLI reports.
Config generatorBuilt-in web UI creates ready-to-use configs for Claude Code, Codex, OpenCode, and Qwen Coder.
Hot reloadConfig reloads on file save or SIGHUP. Add models or rotate keys without restarting.
SecurityConstant-time auth, IP rate limiting, SSRF protection, sanitized error responses, path allowlisting.

Supported backends

Anything that speaks OpenAI Chat Completions or Anthropic Messages protocol works. Tested with:

vLLM llama-server Ollama OpenAI API Anthropic API MiniMax Zhipu (GLM)

How it works

Client request (any protocol) → Protocol handler parses request → Translate to Chat Completions (if needed) → Pipeline: describe images, inject web search tool → Send to backend → Pipeline: execute web search if called, re-send → Translate response back to client protocol → Stream to client

The pipeline is optional. Without processors configured, the proxy just translates and routes. With processors enabled, images and search work transparently on any backend.
